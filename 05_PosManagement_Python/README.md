## 🛒 Smart Mart POS System (Flet 기반 키오스크)
Flet 라이브러리를 활용하여 구축한 실시간 재고 관리 및 매출 정산 POS 시스템이다. 파이썬의 동적 타이핑 특성과 객체 지향적 UI 구성을 실습하기 위해 제작됨.

### 1. 🚀 프로젝트 개요
목적: 대규모 품목(80여 종)을 효율적으로 관리하고 실시간으로 매출을 집계하는 POS 시스템 구현

주요 기술Stack: Python, Flet (GUI Framework), Random (데이터 시뮬레이션)

---

### 2. 🧠 논리 전개 및 설계 과정
- 이 프로젝트는 다음과 같은 단계적 논리 흐름에 따라 설계됨.

    - 데이터베이스 시뮬레이션: 80개의 물품 리스트를 기반으로 random 라이브러리를 활용해 실행 시마다 가변적인 가격과 재고 데이터를 생성하도록 하여 현실감을 부여함.

    - 데이터 구조화: 각 상품 정보를 Dictionary 형태(inventory[item] = [price, stock])로 관리하여, 상품명을 키값으로 가격과 재고에 즉각 접근할 수 있도록 최적화함.

- UI/UX 레이아웃 분리:

    - 좌측 (상품 영역): GridView를 사용하여 많은 양의 데이터를 스크롤 가능한 격자 형태로 배치.

    - 우측 (결제 영역): 장바구니 리스트와 매출 현황을 고정 배치하여 실시간 확인 가능.

상태 업데이트 로직: 상품 클릭 시 재고 차감 -> 장바구니 추가 -> 총액 계산이 동시에 이루어지는 이벤트 기반 프로그래밍을 적용함.

---

### 3. 🏗️ 코드 핵심 개념 및 구조
🔹 데이터 구조 (Data Structure)
데이터 구조 (Data Structure)
Inventory: Dict[str, List[int, int]] 구조를 사용해 O(1)의 시간 복잡도로 상품 정보를 조회함.

Cart: List를 활용해 구매 예정 내역을 순차적으로 담음.

- checkout(e):

    - 매출 집계: nonlocal 키워드를 사용하여 전역 변수인 누적 매출액과 판매 개수를 안전하게 업데이트함.

    - 초기화: 결제 완료 후 장바구니 리스트를 비우고 정산 텍스트를 최신화함.

---

### 4. ⭐ 중요 구현 포인트
- 동적 UI 생성: 80개의 아이템을 일일이 코딩하지 않고, for문을 통해 Container 객체를 자동 생성하여 코드의 효율성을 높임.

- 사용자 피드백: 재고 부족 시 SnackBar 알림을 띄워 사용자 경험(UX)을 개선함.

- 가독성 높은 정산: f-string의 :, 포맷팅을 활용하여 천 단위 콤마(예: 10,000원)를 적용, 시인성을 확보함.
