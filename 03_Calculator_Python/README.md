# 🔢 OOP(객체지향) 기반 단계별 사칙연산 계산기
**상속(Inheritance)**을 통한 견고한 데이터 처리 로직과 Flet 프레임워크 기반의 안정적인 UI를 결합한 파이썬 계산기 애플리케이션.

## 1. ✨ 프로젝트 주요 특징
OOP 상속 구조: 숫자만추출 클래스를 상속받아 연산에 필요한 데이터만 필터링하는 백엔드 로직 설계.

안정적인 UI Layout: 컨테이너의 고정 height 대신 padding과 Row 정렬을 사용하여 화면 확장 시 발생하는 렌더링 버그(회색 영역)를 해결.

유연한 데이터 처리: 가변 인자(*args)와 재귀적 추출 로직을 통해 단일 숫자뿐만 아니라 리스트, 튜플 형태의 입력도 처리 가능.

---
## 2. 🧠 시스템 아키텍처 및 논리 구조
🔹 백엔드: 객체 지향적 설계 (Inheritance)
- 상속 구조: 계산기 클래스는 숫자만추출 클래스를 상속받아 기능의 분리를 실현함.

- 데이터 정제: isinstance와 type 체크를 통해 입력값 중 유효한 int, float 데이터만 추출하여 self.숫자 리스트에 저장함.

- 예외 처리: 나누기 연산 시 0으로 나누는 상황에 대한 방어 로직을 포함하고 있음.

🔹 프론트엔드: 사용자 경험(UX) 최적화
- Step-by-Step UI: 사용자 편의를 위해 입력 과정을 3단계(숫자1 → 연산자 → 숫자2)로 시각화하여 가독성을 높임.

- 방어적 프로그래밍: try-except 구문을 활용해 숫자가 아닌 값이 입력될 경우 즉각적인 오류 메시지를 출력함.

- 레이아웃 최적화:
   - Issue: 고정 높이 설정 시 특정 해상도에서 회색 배경이 나타나는 렌더링 버그 발생.
   - Solution: height 속성을 제거하고 padding을 통해 자연스러운 높이를 확보하여 반응형 대응.

## 3. 🛠️ 핵심 코드 및 개념
- Back-end : 적용기술 |super().__init__()| -> 부모 클래스의 초기화 메서드를 호출하여 상속 관계 확립
- Data : 적용기술 |is_integer()float| -> 결과값이 정수일 경우 .0을 제거하여 깔끔하게 표시
- Front-end : 적용기술 |ft.Dropdown| -> 연산자 선택을 제한하여 사용자 입력 오류 사전 방지
- Formatting : 적용기술 |f"{res:,}| -> 결과값에 천 단위 콤마를 적용하여 시인성 확보
